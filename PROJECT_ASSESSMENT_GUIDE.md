# Software Development Unit (SDU)
## Project Evaluation Rubrics

This document defines the official evaluation criteria for projects submitted under the **Software Development Unit (SDU)**.  
Projects are assessed based on **technical quality, documentation, ethical practices, and real-world impact**, with expectations scaled to each track.

## General Evaluation Rules (All Tracks)

- Plagiarism or blind copying (including AI output) is **not allowed**
- Contributors must understand and be able to explain their code
- Documentation quality is mandatory
- Ethical and responsible use of AI tools is required

---

## Reviewer Guidelines

Reviewers should:
- Apply criteria consistently and fairly
- Focus on learning, growth, and improvement
- Provide clear, constructive, and actionable feedback
